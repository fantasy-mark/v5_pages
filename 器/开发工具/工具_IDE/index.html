    AGP版本不对

    The project is using an incompatible version (AGP 7.3.1) of the Android Gradle plugin. Latest supported version is AGP 7.2.1

    // 这里按照提示把 'com.android.application' 'com.android.library' 版本改为 7.2.1 就解决问题了, 原因可以看以下
// Top-level build file where you can add configuration options common to all sub-projects/modules.
plugins {
    id 'com.android.application' version '7.2.1' apply false
    id 'com.android.library' version '7.2.1' apply false
    id 'org.jetbrains.kotlin.android' version '1.7.10' apply false
}

    这里有三个概念：

    1. AS（Android Studio）版本 - 基本不换IDE
    2. Gradle版本 - 项目包含下载了一般不改
    3. AGP（Android Gradle Plugin）版本 - 配置脚本就好，一般动这个

    所以一般确认更改AGP版本号流程 AS -> Gradle -> AGP

    gradle文档

    首先 Android Studio 有一个支持Gradle插件的范围（查看 Help-About 可见版本），然后从项目 gradle-wrapper.properties 文件看使用的Gradle版本（它有最低的插件）

    可以看到插件版本为 7.2.x时只要gradle版本大于7.3.3即可，as满足条件

    wsl安装

    环境要求
    • Windows 10 以上，版本要求已更新到版本 2004 的内部版本 19041 或更高版本Win11系统
    组件安装
    • 打开 “控制面板”-》“程序”-》“启用或关闭window功能”

    安装wsl命令

    在 PowerShell 或 Windows 命令提示符中输入wsl --install，需要管理员方式运行。

    安装后建议执行wsl --update更新下。 需要重启系统。

    安装ubuntu发行版本

    常用快捷键

    系统：

    1. Alt + Enter：根据提示补全代码
    2. Ctrl + Tab：在不同窗口之间切换
    3. Ctrl + Tab：在不同窗口之间切换

    功能：

    1. Ctrl + N：快速查找类
    2. Ctrl + Shift + N：快速查找文件
    3. Ctrl + Alt + L：对代码进行格式化
    4. Ctrl + /：注释或取消注释
    5. Ctrl + Shift + F：全局查找
    6. Ctrl + Shift + R：全局替换

    设置ssh远程同步

    • Automatic Upload(Alway) - 自动同步本地修改到远程
    • 可以通过RemoteHost中通过Exclude Path来控制是否参与同步
    • 直接登录远程修改文件不会同步到本地，需要手动Download覆盖
